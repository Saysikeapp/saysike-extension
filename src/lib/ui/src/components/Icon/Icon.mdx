# Icon

## Icon location and styles

Icons are hosted on our [CDN](https://cdn.saysike.com) and are accessed by their path, which is specified in the `src` prop of the Icon component.

_Most_ of our icons are coming from the [Orion icon library](https://www.orioniconlibrary.com/).

A select number come from MUI or are custom made where Orion has no alternative.

IMPORTANT - we can only use up to 180 in total from Orion under our license, including variations.

They come in two major groups: `line` and `solid` (`color` and `flat` also exist, but we have very few if any right now). Each group is a folder on the CDN.

When running process icons, all icons (regardles of type) should be in the base folder of icon-input, and each contain a `desc` tag that describes what kind it is, like below:

```tsx
<title>Add To Card</title>
<desc>A (solid/line/color) styled icon from Orion Icon Library.</desc>
```

To display the "agenda" icon in the `solid` style, you can specify a `src` of `solid/add-to-cart.svg`. As most icons used in our designs are of the style `solid`, if you omit the style-folder, it will default to it. `<Icon src="add-to-cart.svg">` is equivalent to `<Icon src="solid/add-to-cart.svg" />`.

## Colours

By default, icons will take the colour of the text node they're in.

<div className="text-primary">
  This text has the primary colour applied to it, as it's inside a div that has
  the "text-primary" class.
  <br />
  This icon will have the same colour: <Icon src="add-to-cart.svg" />
</div>

```tsx
<div className="text-primary">
  This text has the primary colour applied to it, as it's inside a div that has
  the "text-primary" class.
  <br />
  This icon will have the same colour:
  <Icon src="add-to-cart.svg" />
</div>
```

Based on text colour class names, Icons can receive dedicated colour class names: `icon-primary`, `icon-danger`, etc.

Icon component with a css class of `icon-danger`: <Icon src="add-to-cart.svg" className="icon-danger" />

```tsx
<Icon src="add-to-cart.svg" className="icon-danger" />
```

## How to add icons

In the repository, you need to put the **full** list of icons in `./scripts/icons-input/`. The export from Orion's website will be a zip file containing all the icons at the root, without any folder tree.

Then, open a terminal and run `npm run prepare-icons`. This script will do 3 things:

- Dispatch the icons into style-folders (color, flat, line and solid), also adding a css class onto the svg element to mark the style (`icon-solid` for example).
- Generate a type for the src prop of the Input component.
- Generate Javascript array listing all the different possibilities for the `src` prop, used to display all the possible values on this page below.
